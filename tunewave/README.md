# TuneWave — Music Distribution Dashboard

A Flask web dashboard where an artist can upload, manage, and distribute
their music catalog, with a built-in analytics dashboard.

## Features
- **Upload tracks**: title, artist, genre, release date, optional cover art
- **Catalog view**: table of every track with cover thumbnails, genre tags,
  and quick removal
- **Analytics dashboard**:
  - Total tracks, 30-day stream count, and estimated revenue
  - A streams-over-time line chart (last 30 days, aggregated across the
    whole catalog)
  - A "top tracks" leaderboard ranked by recent streams
  - A genre breakdown bar chart
- **Per-track detail page** with its own 30-day streams chart, average
  daily streams, and estimated revenue

## How the analytics work
Since there's no real streaming-platform API to pull from, each track's
daily stream counts are generated by a **deterministic simulation**:
a random walk seeded from the track's database ID, with no streams
before its release date and a gentle upward trend afterwards. The
numbers stay consistent across page loads (so it feels like a real
dataset) while clearly being a working prototype you can swap a real
analytics source into later.

## Running locally

```bash
pip install -r requirements.txt
python app.py
```

Then open http://127.0.0.1:5001 in your browser.

## Project structure
```
tunewave/
├── app.py                  # routes, DB, and analytics simulation
├── requirements.txt
├── static/
│   ├── style.css
│   └── covers/             # uploaded cover art (created automatically)
└── templates/
    ├── base.html
    ├── dashboard.html       # analytics dashboard with charts
    ├── catalog.html         # catalog table
    ├── upload.html          # upload form
    └── track_detail.html    # per-track analytics
```

## Connecting real analytics later
Replace `simulate_daily_streams()` in `app.py` with a call to your
distributor's API (e.g. Spotify for Artists, DistroKid, etc.), returning
the same `[(date, stream_count), ...]` shape — the dashboard and charts
will work unchanged.
